## Analysis

### Summary
In his second consecutive solo episode on AI agents, Dylan shares a detailed week-two progress report on building with OpenClaw as a complete non-developer. He walks through two specific use cases — a fitness tracking/personal trainer agent and a daily news aggregation agent for his livestreams — documenting the real hours invested, the frustrations of data management and agent hallucinations, and the critical role of Obsidian as a "second brain" for organizing agent workspaces.

### Topics Covered
- Week two of OpenClaw journey: "The first week I didn't know anything, but I didn't know that I didn't know anything"
- Use case 1: Fitness and diet tracking agent — personal trainer replacement using 2.5 years of workout data
- Migrating from single-file human-readable logs to individual markdown files for agent consumption
- Data formats: human data vs. agent data — long documents introduce overwrite/deletion risks
- Telegram integration for logging workouts via natural language messages
- Weekly fitness analysis using Opus model; lighter weekly reports from cheaper models
- Use case 2: Daily Digital Download script agent — 24-hour crypto news aggregation and talking points
- Model hopping issues: switched from Qwen to Grok; inconsistent daily report delivery
- The "secret sauce": feeding daily livestream transcripts back to the agent to build a corpus of Dylan's voice and worldview
- Obsidian as "second brain" for managing agent workspaces, vault architecture, and daily notes
- Template overwriting problems: agents destroying new templates, requiring append-only and file locking
- Learning shell scripts and coding through Claude assistance
- Agent daily notes as relationship-building mechanism — two weeks of accumulated context
- The 10x input today for 10x output tomorrow principle
- Hofstadter's law applied to AI agent setup: everything takes longer than expected
- Cost management: local cron jobs vs. paid inference for recurring tasks
- Lessons from non-tech OpenClaw users: homeschool mom designing classes, pastor managing 13,000 contacts
- The shift from "user of agents" to "project manager of agents"
- Concern about pivoting too much toward AI content vs. staying in crypto/blockchain lane
- Counterpoint: everyone is pivoting to AI, so now is the best time to be a crypto content creator
