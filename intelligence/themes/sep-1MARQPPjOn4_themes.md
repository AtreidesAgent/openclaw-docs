# SEP: Making Crypto Feel Normal: Zeymer Montes on MOCA, Payments, & UX That Actually Works

## 1. Mocha Ecosystem Vision
- Description: Mocha aims to be the 'Apple of crypto' – an integrated ecosystem of wallet, POS, messenger, and supporting apps that work seamlessly together for intuitive crypto use.
- Evidence: Multiple interconnected products including wallet, point-of-sale, messenger, and web solutions designed for frictionless user retention.

## 2. UX Prioritization from Web2 Experience
- Description: Lessons from Web2 development emphasize bridging technical implementation with user experience, QA, and client understanding to create intuitive products.
- Evidence: Founder's background managing 15 developers, focusing on products simple enough for non-technical users like 'my mom can use it.'

## 3. Onboarding Friction Reduction
- Description: Crypto onboarding challenges like seed phrases and technical jargon alienate users; solutions involve familiar terminology (e.g., 'account number' instead of address).
- Evidence: Replacing 'address' with 'account number,' 'transaction hash' with 'transaction ID/hash' for broader accessibility.

## 4. Regional Event Insights (ETH Mexico)
- Description: Events like ETH Mexico build local communities by using native languages (Spanish), attracting students and non-English speakers to crypto concepts.
- Evidence: Praise for student attendance, native-language talks reducing barriers for newcomers vs. English-dominant global events.

## 5. Merchant POS Simplicity
- Description: POS interface mimics familiar credit card terminals – enter amount, generate QR, show to customer – enabling instant usability without training.
- Evidence: Live tests at events where users (young/old) correctly charged amounts in seconds without instructions.

## 6. Universal Wallet Compatibility
- Description: 'Pay with any wallet' via QR-to-web flow: scan opens branded payment page selecting token/address/amount, no app required.
- Evidence: Works with phone camera; supports major wallets with prefill; any phone, any supported token.

## 7. Cross-Chain Atomic Swaps
- Description: Users pay in preferred token (e.g., BTC/SOL); merchant receives desired stablecoin (USDC/USDT) via decentralized swaps.
- Evidence: Backend uses THORChain, Maya, etc.; listens to mempool for trustless conversion with confirmation rules by amount.

## 8. Self-Custody Emphasis
- Description: No wallet connections or permissions; users copy-paste address/amount manually for maximum privacy and security.
- Evidence: Disclaimers, copy buttons, no dApp approvals to minimize Mocha's liability and attack surface.

## 9. MPC Recovery for Normies
- Description: Optional multi-party computation recovery: user holds shares; Mocha encrypts one share (PIN-required) anonymously for non-tech users.
- Evidence: Separate database, brute-force resistant; defaults to saving for new users without manual seed handling.

## 10. Matrix Protocol for Messenger
- Description: Secure, encrypted E2E messaging with instant decryption on-device; push notifications trigger background decrypt for previews.
- Evidence: Overcame challenges for real-time feel despite encryption; aims for 95% non-crypto users first.

## 11. Lifetime Referral Revenue
- Description: Perpetual revenue share (e.g., 20bps) from referred users/businesses' Mocha transactions, carved in stone.
- Evidence: Applies to wallets, POS businesses (e.g., onboard Starbucks = lifelong cut); incentivizes early BD swarm.

## 12. Beta Hubs Strategy
- Description: Seed mid-sized cities with signage/partners for physical/digital ubiquity, building trust via omnipresence before global scale.
- Evidence: Local leverage for POS onboarding; 'crypto accepted' signs like Visa/Mastercard.

## 13. Web-First Accessibility
- Description: All products web-accessible to bypass app store compliance/censorship in restricted countries.
- Evidence: Wallet/POS on web; no downloads needed.

## 14. Multi-Chain Explorer Integration
- Description: Unified explorer links from all products for transaction verification across chains.
- Evidence: Instant tx details from POS/messenger/wallet.

## 15. Normie-First Language
- Description: Avoid jargon; use banking metaphors (account, transaction ID) to onboard older generations/tech-reluctant users.
- Evidence: ETH Mexico audience question on teaching elders; floppy disk 'save' icon analogy.

## 16. Decentralized Data Future
- Description: Phase 2 messenger shifts to decentralized DB for messages once volume justifies; reduces central liability.
- Evidence: Matrix backend already; full WeChat-like payments planned.

## 17. Error-Proof Flows
- Description: Bulletproof UX with visual confirmations (processing/accepted checkmarks), amount/address disclaimers, auto-copy.
- Evidence: Mempool trust rules scale by tx size (instant small, confirm large).

## 18. Long-Term Builder Mindset
- Description: Stealth build for 2 years; demo working products to investors before funding to prove traction.
- Evidence: June/July POS/wallet push; end-year messenger payments.

## 19. Latin America Payments Context
- Description: QR/mobile payments normative (vs. NFC taps in West); leverages existing habits for crypto adoption.
- Evidence: Street vendors/businesses intuitively handle QR flows.

## 20. Inclusive Technology Ethos
- Description: Technology must improve all lives, not isolate groups; accessible UX prevents exclusion of non-tech-savvy/older users.
- Evidence: Native languages at events, grandma-friendly POS, recovery defaults.
